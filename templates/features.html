Bag of Words

CountVectorizer · max_features = 5,000

Bag of Words (BoW) counts how many times each word appears in a document. Each unique word across the corpus becomes a column in the feature matrix — and each row is a review represented as a vector of word counts.

How it works
  1. Build a vocabulary from the entire training corpus.
  2. Limit to the top 5,000 most frequent words.
  3. Represent each review as a count vector over this vocabulary.
Example
# Review: "great film great acting"
vocabulary = ["great", "film", "acting", "boring", ...]
vector = [ 2, 1, 1, 0, ...]
Pros
  • Simple and fast to compute
  • Works well with Naive Bayes
  • Captures word frequency
Cons
  • Ignores word importance
  • Common words dominate
  • No semantic meaning

TF-IDF

TfidfVectorizer · max_features = 5,000

Term Frequency–Inverse Document Frequency weighs each word by how often it appears in a document relative to how rare it is across all documents. Common words like "the" get low scores; distinctive words get high scores.

Formula
TF-IDF(t, d) = TF(t, d) × IDF(t)
TF(t,d) = count(t,d) / total_words(d)
IDF(t) = log( N / df(t) ) + 1
How it works
  1. Compute term frequency (TF) for each word in each document.
  2. Compute inverse document frequency (IDF) across the corpus.
  3. Multiply TF × IDF → higher scores for rare, informative words.
Pros
  • Penalises common stop-words
  • Highlights unique terms
  • Better for LR / Neural Nets
Cons
  • Slightly more expensive
  • Still no word order
  • Sparse matrix output
